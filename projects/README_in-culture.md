## What it does

Signal is a single-page web tool for brand strategists and creatives. You paste a URL, optionally name a brand, and Signal returns:

1. **Résumé** — a concise cultural analysis of the article
2. **Shower Thought** — the sharpest universal truth hidden in the piece
3. **Idée créative** — a "Et si [marque]…" brand opportunity built on that truth

Everything runs in French. Everything happens in the browser — no database, no pipeline, no stored data.

## How the pipeline works

```
1. SCRAPE   URL → full article text → AI extraction (title, summary_long) in French

2. SHOWER THOUGHT
   - Semantic search against shower_thoughts index (≈3 000 cultural truths)
   - LLM generates 5 candidate shower thoughts from article + closest matches
   - Evaluator scores each on Simplicity × Surprise × Depth
   - Best one selected

3. CREATIVE IDEA
   - LLM picks 3 creative prismes from a library of 20 frameworks
   - Generates one "Et si [marque]…" opportunity per prisme
   - Evaluator scores each on Truth Response × Strategic Fit × Generativity × Provocativeness
   - Best one returned

4. STREAM   Results are streamed step-by-step via Server-Sent Events:
            article → shower_thought → creative_idea
```

Results are streamed progressively — the résumé appears as soon as the article is scraped, without waiting for the AI steps.

## Brand context (optional)

| Field | Effect |
|---|---|
| **Marque** | Brand name used in creative prompts |
| **Territoire de marque** | Brand territory fed into all generation and scoring prompts. Auto-generated by AI if left empty (blur the field). |
| **Documents de marque** | PDF / DOCX / TXT files — chunked, embedded, and searched in-memory per request. Used to ground the shower thought and creative idea in real brand context. |

If no brand is provided, Signal generates a universal creative opportunity applicable to any brand.

---

## Redo buttons

Each output section has a ↻ button that re-runs only what's needed:

| Button | Re-runs |
|---|---|
| ↻ Résumé | Full pipeline from scraping |
| ↻ Shower Thought | Shower thought + creative idea (keeps article) |
| ↻ Idée créative | Creative idea only (keeps shower thought) |

## Quick start

```bash
# Install dependencies
pip install -r requirements.txt

# Set environment variables
cp .env.example .env
# Edit .env — minimum required: ANTHROPIC_API_KEY and HF_API_TOKEN

# Start the server
uvicorn api.app:app --reload

# Open in browser
open http://localhost:8000
```

---

## Environment variables

| Variable | Required | Description |
|---|---|---|
| `ANTHROPIC_API_KEY` | Yes | Claude API key (article extraction, shower thoughts, creative ideas) |
| `HF_API_TOKEN` | Yes | HuggingFace Inference API token (sentence embeddings) |
| `APP_PASSWORD` | No | Password to protect the app (leave empty to disable auth) |
| `SECRET_KEY` | No | Secret for HMAC-signed session cookie (defaults to `change-me-in-production`) |

## API endpoints

### `POST /process/stream`

Main endpoint. Returns a Server-Sent Events stream.

**Form fields:**
| Field | Required | Description |
|---|---|---|
| `url` | Yes | Article URL to analyse |
| `brand` | No | Brand name |
| `brand_territory` | No | Brand territory description |
| `files` | No | Brand documents (PDF, DOCX, TXT) |

**SSE events emitted:**
```
data: {"type": "article",        "title": "...", "summary": "...", "url": "..."}
data: {"type": "shower_thought", "shower_thought": "..."}
data: {"type": "creative_idea",  "creative_idea": {"tool": "...", "concept": "..."}}
data: {"type": "done"}
data: {"type": "error",          "message": "..."}
```

### `POST /api/territory`

Auto-generate a brand territory from a brand name.

**Form fields:** `brand`
**Response:** `{"territory": "..."}`

### `POST /api/redo/shower_thought/stream`

Re-run shower thought + creative idea from a pre-extracted article.

**Form fields:** `title`, `summary_long`, `url`, `brand`, `brand_territory`, `files`
**SSE events:** `shower_thought`, `creative_idea`, `done`

### `POST /api/redo/idea/stream`

Re-run creative idea only, keeping the existing shower thought.

**Form fields:** `title`, `summary_long`, `url`, `shower_thought`, `brand`, `brand_territory`, `files`
**SSE events:** `creative_idea`, `done`

### `GET /health`

```json
{"ok": true}
```

## Authentication

Auth is disabled by default. To enable it, set `APP_PASSWORD` in your environment. Users will be prompted for the password on first visit; a signed cookie persists the session.

To disable auth again, unset `APP_PASSWORD`.
